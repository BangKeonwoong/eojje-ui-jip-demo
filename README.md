# 어제의집 프로토타입 — 미리보기 / 배포 안내

> 이 프로젝트는 체험용 정적 데모입니다. 실제 인증, 결제, 주문 전송, 카카오 연동, 관리자 권한 통제가 없으므로 운영 데이터를 입력하거나 실제 상점으로 사용하면 안 됩니다.

`index.html` 과 `admin.html` 두 파일이 **각각 완전 독립형(self-contained)** 입니다.
리액트와 실행 런타임이 파일 안에 모두 포함되어 있어, 외부 CDN 로딩 실패로
모바일에서 화면이 비거나 깨지던 문제를 해결했습니다.

- `index.html` — 스토어(홈·컬렉션·상세·매입·장바구니·주문·로그인·마이·찜·이용안내·검색). 모바일 반응형(햄버거 메뉴·하단 탭바).
- `admin.html` — 인증 없는 관리자 데모(대시보드·상품·주문·매입 신청·입고 알림·문의). 검색 엔진 노출 차단 메타가 적용됩니다.
- `accessibility.js` — 키보드 조작, 포커스, 대화상자, 폼 레이블, 이미지 오류 대체 처리.
- `_headers` — Netlify 등 지원 호스트의 CSP·클릭재킹·권한 정책. GitHub Pages는 이 파일을 적용하지 않지만, 핵심 CSP는 두 HTML의 `<meta>`에도 포함됩니다.
- 서로 링크됨: 스토어 푸터 **관리자 데모 ↗** ↔ 관리자 상단 **스토어 보기 ↗**.

## 모바일에서 열 수 있는 공개 링크 (권장, ~30초)
1. 브라우저에서 https://app.netlify.com/drop 접속
2. 이 `web` 폴더를 통째로 창에 끌어다 놓기 (또는 첨부된 zip 을 풀어서)
3. 생성된 `https://…netlify.app` 링크를 휴대폰으로 전송 → 그대로 열림
4. 관리자 데모는 그 링크 뒤에 `/admin.html`

호스팅을 권장하는 이유: 두 페이지 간 이동(관리자 ↔ 스토어)과 안정적 로딩을 위해서입니다.

## 같은 와이파이에서 내 폰으로 미리보기
이 폴더에서:

    python3 -m http.server 8000

휴대폰 브라우저에서 `http://<컴퓨터-IP>:8000` (어드민: `.../admin.html`).

## 데이터와 보안 경계
- 이미지·폰트만 인터넷에서 불러옵니다(없어도 레이아웃은 정상, 이미지 자리만 빈 상태).
- 테마·장바구니·찜은 `localStorage`에, 이름·연락처·데모 주문은 현재 탭의 `sessionStorage`에만 저장됩니다.
- 관리자 화면의 역할 선택은 화면 체험용이며 보안 권한이 아닙니다.
- 현재 런타임은 템플릿 평가를 위해 `unsafe-eval` 예외에 의존합니다. 실서비스 전환 전에 정적 빌드로 이전해 이 예외를 제거해야 합니다.
- 결제·카카오 로그인·사진 업로드·지도 등은 "프로토타입" 목업 흐름입니다.
